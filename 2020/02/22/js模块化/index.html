js模块化
发布于：2020-02-22 20:23:24 标签：/ 学习笔记 / javascript / 访问：

模块化开发是一种管理方式，是一种生产方式，一种解决问题的方案，一个模块就是实现特定功能的文件，有了模块，我们就可以更方便地使用别人的代码，想要什么功能，就加载什么模块。模块化开发需要遵循规范。

模块化的好处:

  • 避免命名冲突(减少命名空间污染)
  • 更高的复用性与更高的可维护性

CommonJS-Node

主要争对服务器端

说明:

  • 每个文件都可当作一个模块，每个文件都有自己的一个作用域
  • 服务器端:模块的加载是运行时同步加载的
    CommonJS需要通过赋值的方式来获取匿名函数自调用的返回值，所以require函数在加载模块是同步的。所以局限了CommonJS在浏览器端上的使用。
  • 浏览器端:模块需要提前编译打包处理

基本语法

暴露模块:
module.exports = value
exports.xxx = value

暴露的本质是exports对象

引入模块：require(xxx)
第三方模块:xxx为模块名
自定义模块:xxx为模块文件路径

实现：

  • 服务器端Node.js
  • 浏览器端：Browserify(编译打包工具)

NodeJS是CommonJS规范的实现，webpack 也是以CommonJS的形式来书写
