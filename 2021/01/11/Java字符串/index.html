String不同拼接操作的对比

  • 常量与常量的拼接结果在常量池，且常量池中不会存在相同内容的常量。
  • 只要其中有一个是变量，就相当于new，结果在堆空间中，注意加了final关键字的变量其实是常量，存储在常量池中。
  • 如果拼接的结果调用intern()方法,返回值就在常量池中。
String s1 = "javaEE";
String s2 = "hadoop";

//s3与s4是字面量从常量池中去找
String s3 = "javaEEhadoop";
String s4 = "javaEE" +"hadoop"; //字面量连接
//有变量名参与，相当于new，在堆空间去创建
String s5 = s1 + "hadoop";
String s6 = "javaEE" + s2;
String s7 = s5.intern(); //返回值得到的s7在常量池中

System.out.println(s3 == s4);//true
System.out.println(s3 == s5);//false
System.out.println(s3 == s6);//false
System.out.println(s5 == s6);//false
System.out.println(s3 == s7);//true

final String s8 ="javaEE"; //final存储在常量池中
String s9 = s8 + "hadoop";
System.out.println(s3 == s9); //true

StringBuffer/StringBuilder类

三者关系
String:不可变的字符序列，底层用char[]数组存储
StringBuffer:可变的字符序列,线程安全的，效率低，底层用char[]数组存储,JDK9.0后是byte[],建议指定参数，避免频繁扩容复制。
StringBuilder:可变的字符序列,底层用char[]数组存储,线程不安全，效率高，JDK5.0新增,建议指定参数，避免频繁扩容复制。

源码分析：

StringBuffer/StringBuilder方法

不同于String的额外方法。

stringBuffer.append(“xx”):用于进行字符串拼接，直接在原字符串上操
stringBuffer.delete(int start,int end):删除指定位置的内容，直接在原字符操作。只要涉及下标位置的都是左闭右开，也就是下标的左边开始划分
stringBuffer.replace(int start,int end,String str):把[start,end)的位置替换成str
stringBuffer.insert(int offset,xxx):在offset位置插入xxx
stringBuffer.reverse():把当前字符串逆转

方式二：采用字符串拼接 -> 优化版本

public String reverse(String str,int startIndex,,int endIndex){
    String reverseStr = str.substring(0,startIndex);
    for(int i = endIndex;i>=startIndex;i--){
        reverseStr += str.charAt(i);//不断创建新的
    }
    reverseStr += str.substring(endIndex+1);
    return reverseStr;
    //使用StringBuilder替换String
    //优化写法
    StringBuilder builder = new StringBuilder(str.length);
    builder.append (str.substring(0,startIndex));
    for(int i = endIndex;i>=startIndex;i--){
        builder.append (str.charAt(i));
    }
    builder.append (str.substring(endIndex+1));
    return builder.toString();
}
