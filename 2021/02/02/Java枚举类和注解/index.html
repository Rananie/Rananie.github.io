Java枚举类和注解

枚举类

定义:类的对象有限个，确定的，我们称此类为枚举类。
使用:当需要定义一组常量时，强烈使用枚举类。
说明：若枚举只有一个对象，则可以作为一种单例模式的实现方式。

如何自定义枚举类？

方式一：jdk5.0之前，自定义枚举类
方式二：jdk5.0,可以使用enum关键字定义枚举类

//自定义枚举类
class Season{
    //1.声明Season对象的属性，类是常量，属性也是常量
    private final String seasonName;
    private final String seasonDesc;
    //2.私有化构造器，因为类的对象是有限个，确定的
    private Season(seasonName,seasonDesc){
        //给对象属性赋值，final的变量不可以使用默认值
        this.seasonName = seasonName;
        this.seasonDesc = seasonDesc;
    }
    //3.提供当前枚举类的多个对象
    public static final Season SPRING = new Season("春天"，"春暖花开");
    public static final Season SUMMER = new Season("夏天"，"夏日炎炎");
    public static final Season AUTUMN = new Season("秋天"，"秋高气爽");
    public static final Season WINTER = new Season("冬天"，"冬天雪地");
    //其他诉求1：get类的属性
    //其他诉求2：提供toString方法
}

如何使用关键字enum定义枚举类？

1.eunm定义的枚举类默认继承于java.lang.Eunm类,该类已重写toString方法,返回的是常量对象的名字SUMMER、SPRING等。
2.开头上来就需要提供当前枚举类的常量对象。

//使用enum关键字
enum Season{
    //1.提供当前枚举类的对象，多个对象之间用","，最后一个对象用";"
   SPRING("春天"，"春暖花开"),
   SUMMER("夏天"，"夏日炎炎"),
   AUTUMN("秋天"，"秋高气爽"),
   WINTER("冬天"，"冬天雪地");

//--------其余没变化，toString已被重写

    //1.声明Season对象的属性，类是常量，属性也是常量
    private final String seasonName;
    private final String seasonDesc;
    //2.私有化构造器，因为类的对象是有限个，确定的
    private Season(seasonName,seasonDesc){
        //给对象属性赋值，final的变量不可以使用默认值
        this.seasonName = seasonName;
        this.seasonDesc = seasonDesc;
    }
    //其他诉求1：get类的属性
}

Enum类的常用方法

枚举类.values():返回一个数组，数组的每个元素是枚举类对象的常量名。
枚举类.valueOf(String str):可以把一个字符串转为对应的枚举类对象名，返回对应的枚举类对象。如果objName没有对应的枚举类对象名,则会抛出异常。
枚举类实例.toString():返回当前枚举类对象常量的名称

实现接口的枚举类

情况1：实现接口，在enum类中实现抽象方法 –> 所有对象共用一个抽象方法
情况2：不同的常量对象分别实现抽象方法 –> 不同对象对该抽象方法的实现不一样

//情况1：实现接口，在enum类中实现抽象方法
enum Seanson implements 接口{
    @Override
}
//情况2：不同的常量对象分别实现抽象方法
enum Season{
   SPRING("春天"，"春暖花开"){
       @Override
   },
   SUMMER("夏天"，"夏日炎炎"){
       @Override
   },
   AUTUMN("秋天"，"秋高气爽"){
       @Override
   },
   WINTER("冬天"，"冬天雪地"){
       @Override
   };
}

注解(Annotation)

Annotation其实就是代码里的特殊标记，如@Override。这些标记可以在编译，类加载、运行时被读取，并执行相应的处理。通过Annotation，程序员可以在不改变原有逻辑的情况下，在源文件中嵌入一些补充信息。代码分析工具、开发工具和部署工具可以通过这些补充信息进行验证或部署。

Annotation可以像修饰符一样被使用，可用于修饰包、类、构造器、方法、成员变量、参数、局部变量的声明，这些信息被保存在Annotation的”name=value”对中。

注解是一种趋势，一定程度可以说：框架 = 注解 + 反射 + 设计模式

常见的Annotation示例

使用Annotation时要在其前面增加@符号,并把该Annotation当成一个修饰符使用。

示例一：生成文档相关的注释
@author 标明开发该类模块的作者，多个作者之间使用,分割
@version 表明该类模块的版本
@see 参考转向，也是相关的主题
@date 编写的时间
以下三个标记只适用于方法
@param 形参名 形参类型 返回值说明
@return 返回值类型 返回值说明
@exception 异常类型 异常说明

示例二:在编译时进行格式检查(JDK内置的三个基本注解)
@Override:限定重写父类方法，该注解只能用于方法
@Deprecated:用于表示锁修饰的元素(类、方法等)已过时。
@SuppressWarnings({“警告信息1”,”警告信息2”}):抑制编译器警告

示例三：跟踪代码依赖性，实现替代配置文件功能。
Servlet3.0提供了注解，使得不在需要在web.xml文件中进行Servlet的部署
spring框架中关于”事务”的管理 @Text

自定义Annotation

参照SuppressWarnings定义

Annotation的成员变量在Annotation定义中以无参数方法的形式来声明。
其方法名和返回值定义了该成员的名字和类型，我们称为配置参数。
类型只能是八种基本数据类型、String类型、Class类型、enum类型、Annotation类型以及以上所有类型的数组。
如果只有一个参数成员,建议使用参数名为value。
定义Annotation的成员变量时为其指定初始值，指定成员变量的初始值可使用default关键字。
没有成员定义的Annotation称为标记，包含成员变量的Annotation称为元数据Annotation。

public @interface MyAnnotation{
 String value();
 //可以指定默认值
 String value () default "hello";
}
//使用时
@MyAnnotation(value = "hello")
//当只有一个成员时，可以省略成员名
@MyAnnotation("hello")

JDK中的元注解

定义：元注解用于修饰其他Annotation定义。

JDK5.0提供了4个标准的meta-annotation类型，分别是Retention,Target,Documented,Inherited。

元数据的理解：String name = “ranan”,真实数据是ranan,String name可以看成对ranan的修饰，元数据：对现有的数据进行修饰

@Retention:指定修饰的Annotation的生命周期，包含了RetentionPolicy类型的成员变量
RetentionPolicy.SOURCE:在源文件中有效(源文件保留)，编译器直接丢弃这种策略的注释。
RetentionPolicy.CLASS:在class文件中有效(class保留)，当运行Java程序时，JVM不会保留注释，这是默认值。
RetentionPolicy.RUNTIME:在运行时有效(运行时保留),当运行Java程序时，JVM会保留注释，程序可以通过反射获取该注释。

@Target:用于指定被修饰的Annotation能用于修饰哪些程序元素
TYPE:Class，interface
FIELD:文件
METHOD:方法
PARAMETER:参数
CONSTRUCTOR:构造器
LOCAL_VARIABLE:局部变量
ANNOTATION_TYPE:注解类型
PACKAGE:包

以下两个注解使用频率低
@Documented:用于指定被修饰的Annotation类将被javadoc工具解析时保留下来。默认情况下Javadoc是不包括注解的。
@Inherited:被修饰的Anotation将具有继承性。如果某个类使用了被@Inherited修饰的Annotaiton，则其子类将自动具有该Annotaiton。

JDK8 中注释的新特性

可重复注解

注解可以重复定义多个

1.在MyAnnotation声明@Repeatable(MyAnnotations.class)。关联两个注解
2.MyAnnotation的Target、Retention、Inherited和MyAnnotations相同

//jdk8
@Repeatable(MyAnnotations.class)
public @interface @MyAnnotation{
    String value() default "hello";
}
@MyAnnotation(value = "hello")
@MyAnnotation(value = "abc")
//jdk 8 之前的写法
@MyAnnotations({@MyAnnotation(value = "hello")}，{@MyAnnotation(value = "abc")})

public @interface @MyAnnotations{
   MyAnnotation[] value(); 
}

类型注解

TYPE_PARAMETER:表示该注解能写在类型变量的声明语句中(如泛型声明)
TYPE_USE:表示该注解能写在使用类型的任何语句中。

@Target({TYPE_PARAMETER,TYPE_USE})
public @interface @MyAnnotation{}
//使用
class Generic<@MyAnnotation T>{
    public void show(){
        ArrayList<@MyAnnotation String> list = new ArrayList<>()
    }
}
