c++笔记

编译

常见的预编译和条件编译指令

#define //定义一个预处理宏
#undef  //取消宏的定义
#if     //编译预处理中的条件命令，相当于C语法中的if语句
#ifdef  //判断某个宏是否被定义，若已定义，执行随后的语句
#ifndef //与#ifdef相反，判断某个宏是否未被定义
#elif   //若#if, #ifdef, #ifndef或前面的#elif条件不满足，则执行#elif之后的语句，相当于C语法中的else-if
#else   //与#if, #ifdef, #ifndef对应, 若这些条件不满足，则执行#else之后的语句，相当于C语法中的else
#endif  //#if, #ifdef, #ifndef这些条件命令的结束标志.

include

说明:include是C++的编译预处理命令,作用为引入对应的文件

预处理器处理程序的源代码，是在编译器之前运行。

#include只接受一个参数：头文件名

include的两种写法区别

include<***.h>表示让编译器在编译器的预设标准路径下搜索响应的头文件，如果找不到就报错。
include"***.h"表示先在工程路径下搜索，如果失败，再到系统标准路径下搜索

总结：标准库文件两种方式都可以,用户自定义的头文件只能采用””的方式

#include <iostream> //效率较高
#include "iostream" //效率较低

条件编译

定义:一般情况下，源程序中所有的行都参加编译。希望当满足某条件时对一组语句进行编译，而当条件不满足时则编译另一组语句。

常见形式

形式1说明:当标识符已经被#define命令定义过，则对程序段1进行编译，否则编译程序段2。其中#else可以省略。

#ifdef 标识符
程序段1
#else
程序段2
#endif

形式2说明:如果标识符未被#define命令定义过，则对程序片段1进行编译；否则对程序片段2进行编译。

#ifndef 标识符
程序段1
#else
程序段2
#endif

形式3说明:如果表达式的值为真，则对程序片段1进行编译；否则对程序片段2进行编译。因此可以使程序在不同条件下完成不同的功能。

#if 标识符
程序段1
#else
程序段2
#endif

常见使用

最常见的就是头文件使用的防止重复定义的写法。
说明:编译时把所有文件包含起来，第一次编译到此处时，会定义_ABCD_H_并编译下面的代码。往下继续编译时，倘若有其它文件引用了这个头文件，会再次遇到这一段代码，然后发现_ADCD_H_已经被定义，所以“ifndef _ADCD_H_”后面的代码都不会编译，这样就防止了头文件的重复定义。

#ifndef _ABCD_H_
#define _ABCD_H_

/*
......
*/

#endif

作用域

::表示作用域和所属关系。

作用域符号

为了避免不同的类有名称相同的成员而采用作用域的方式进行区分。或者在类里声明了,并没有定义，在类外定义、
格式: 类名称::类的成员名称
举例:int cbmc_parseoptionst::doit()
说明:cbmc_parseoptionst类里的int doit()方法

全局作用域符号

当全局变量在局部函数中与其中某个变量重名，那么就可以用::来区分此时使用的是全局变量。

命名空间作用域符

用来注明所使用的类、函数属于哪一个命名空间的
格式:namespace::name

构造函数与析构函数

析构函数:当对象被销毁时被系统调用的方法，一般用来清空内存。

class Line{
    Line();  //构造函数声明
    ~Line(); //析构函数声明
